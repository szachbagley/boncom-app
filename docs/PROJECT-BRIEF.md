## The Build
A bit of context. You've been spending your first weeks at Boncom meeting different teams, getting a feel for how everyone works. One of them walks you through how they handle client estimates: everything lives in spreadsheets, with line items copied around by hand, formulas rebuilt for every new quote, and tax math that has to be checked by hand every time. Somewhere in the middle of that walkthrough you think, why don't I just build something that handles this?

That's the app you're building.

## What to build
Build a small web app that lets a user create and manage cost estimates for a client. At minimum, a user should be able to:

- Create an estimate for a named client
- Add multiple line items to it (for example: description, quantity, rate)
- See the totals update live as they edit
- Apply tax and a discount
- Give the estimate a status (for example: draft vs. sent) and come back to it later
 

Beyond the essentials, the shape of the app is yours. How you model the data, where the math lives, what the UI feels like, what you name it, and what you build first are all decisions we want to see you make.

Since someone would actually use this every day, how it feels to use matters to us. We're a creative agency, so thoughtful design and user experience count for a lot, not only the code underneath.
